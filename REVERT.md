## How to revert

```bash
# 1. Code ‚Äî return to the checkpoint (drops all refactor commits)
git checkout design-revamp            # or:  git reset --hard ba9aaaa

# 2. Dependencies ‚Äî restores the checkpoint lockfile (removes Tiptap, etc.)
pnpm install

# 3. Database ‚Äî rebuild the dev DB from the checkpoint's migrations + seed.
#    Restores products.specs, product_types and spec_schema; drops the new
#    columns. (Dev DB only ‚Äî this wipes + re-seeds.)

# 4. (optional) delete the work branch
git branch -D mw-content-model
```

Because git restores the migration files and `seed.sql` to the checkpoint state,
`supabase db reset` reproduces the original schema exactly ‚Äî no hand-written rollback SQL
needed. Generated types (`packages/db/src/database.types.ts`) are also restored by git.
